# Visual Style Guide

This guide distills the desired layout grammar from the user's style-learning process. It is written as text rules so the public Skill can be open-sourced without redistributing third-party reference images.

If local private style reference images exist in `assets/style-references/`, they may be used on the owner's machine. Do not require those images for public operation.

## Aspect Ratio Rule

Always output `9:16` by default. Preserve the scrapbook layout language even if reference images use other vertical social ratios.

## Core Positioning

The desired image is a practical travel-flow scrapbook:

- Route map plus sticker collage.
- Dense but readable phone layout.
- Photo stickers mixed with hand-drawn landmark/food/transport icons.
- Food, transport, hotel, attraction, ticket, weather, and reminder nodes.
- Character stickers as atmosphere, not the main content.

The page should answer quickly:

`今天去哪 -> 怎么走 -> 哪些点必看/必吃 -> 什么时候必须动 -> 天气/票/交通要注意什么 -> 哪些图可以替换`

## Layout Grammar

- Use a large rounded `Day X` badge near the top as the visual entry.
- Let the route line travel through the whole page: dashed gray line, curved road, arrows, or station chips.
- Place the main route nodes along that line, not in random collage order.
- Use overlapping paper labels, small speech marks, arrows, and doodle lines to connect stickers.
- Mix image sizes: one or two large anchor visuals, several medium photo stickers, and smaller food/detail stickers.
- Keep important cards upright enough to read; playful tilt is for photos and decorative labels.
- For merged days, split the page into two readable bands while keeping one connected route story.

## Visual Language

- Beige or light cream grid-paper background.
- Thick white-border cutout photo stickers.
- Hand-drawn black doodles and landmark illustrations.
- Rounded color tabs and speech labels.
- Soft blue and muted brick red for major Day/title blocks.
- Cream yellow labels for place notes.
- Gray dashed route line as the navigation backbone.
- Black handwritten labels; Chinese output should remain Chinese.

## Information Hierarchy

1. Day badge and city/theme.
2. Route line and ordered nodes.
3. Hard transport/time/ticket/weather cards.
4. Hotel and address cards.
5. Attraction/food photo stickers.
6. Reminder bubbles.
7. Character stickers and decorative doodles.

Do not let decorative stickers outrank route, time, hotel, weather, or transport information.

## Text Rules

- Use short handwritten Chinese labels, not paragraphs.
- Compress each node to `地点名 + 动作/用途 + 时间/提醒`.
- Use labels such as `先入住`, `三点前离开`, `门票115`, `晚上动车回市区`, `建议补充`, `雨天备选`.
- Keep route-critical text larger than decorative comments.

## Image And Sticker Rules

- Each scenic/food/hotel/transport visual should be its own white-border sticker.
- Keep enough separation so photo stickers can become replaceable slots.
- Use a hand-drawn icon when a real photo is unavailable, but label it as a visual suggestion if it is not user-provided.
- Prioritize route usefulness over a prettier collage.

## Character Rules

- Use user-uploaded or local private characters only when available.
- Otherwise use generic traveler companion stickers.
- Place characters in corners, beside reminder bubbles, or near route nodes.
- Do not let characters cover route lines, time cards, weather cards, or replaceable image slots.
- Public prompts must not mention private character names or distinctive private IP traits unless the user supplied those references.

## Negative Rules

- Do not create a route-less collage.
- Do not hide flight/train times in tiny handwritten text.
- Do not invent booked hotels, trains, tickets, or places.
- Do not present supplemented visuals as confirmed itinerary facts.
- Do not switch the default aspect ratio away from 9:16.
- Do not redistribute third-party style reference images in the public Skill unless they are explicitly licensed for redistribution.
