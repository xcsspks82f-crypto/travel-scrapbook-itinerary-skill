# 旅行流程手账生成器 Skill

这是一个 Codex Skill，用来把旅行计划、截图 OCR、表格行程，或只有目的地和想法的粗略旅行需求，转成适合 AI 生图的 `9:16` 竖版旅行流程可视化手账。

它不是普通旅游攻略生成器。核心目标是：旅行当天打开图片后，能快速看懂路线、时间、交通、酒店、门票、天气、餐饮和提醒。

## 能做什么

- 解析旅行行程：城市、日期、交通、酒店、景点、美食、门票、时间卡点和备注。
- 判断行程可行性：识别偏赶、偏空、酒店切换、跨城交通和天气风险。
- 自动补齐规划：从粗略想法生成路线、住宿区域、交通方式、小吃推荐和雨天备选。
- 实时信息优先：天气、交通、开放时间、餐饮状态、酒店价格等默认需要实时查询或标注待复核。
- 智能拆页：不是机械每天一页，而是按信息密度决定单页或合并页。
- 生成手账提示词：每页输出可复制到生图模型的完整提示词。
- 支持直接生图：有图片生成工具时，默认每页生成 1 张 `9:16` 图片。
- 支持换图：每页会给照片贴纸分配稳定槽位，例如 `P1-IMG3`，方便后续替换景点、美食、酒店或交通图片。

## 典型输入

### 完整行程

```text
5月24日 上海到苏州
上午去平江路做旗袍妆造，下午去狮子林，晚上去金鸡湖看夜景。
5月25日 睡到中午回上海。
```

### 粗略想法

```text
我2026年5月24日~5月25日想从上海来苏州玩一天一夜。
第一天想体验苏式园林拍旗袍照，做手推波发型，晚上看金鸡湖夜景。
第二天睡到中午回上海，帮我做旅行手账图。
```

### 截图 / OCR / 表格

把行程截图识别后的文字、备忘录内容、Excel 表格内容直接粘贴给 Codex 即可。

## 默认输出结构

Skill 会按以下顺序输出：

```text
【1. 行程结构化拆解】
【2. 手账页数建议】
【3. 每页内容规划】
【4. 每页生图提示词】
【5. 批量生成说明】
```

如果是粗略想法，还会先补充：

```text
【基础假设与实时查询】
【行程可行性判断】
【天气与穿搭/携带提醒】
【推荐行程安排】
【交通方式推荐】
【住宿区域建议】
【景点附近小吃/餐饮建议】
```

## 生图风格

默认视觉方向：

- 9:16 竖版
- 米白方格纸背景
- 旅行手账拼贴
- 白色描边照片贴纸
- 灰色虚线路线
- 黑色手写感中文标注
- 大号 Day 日期气泡
- 景点 / 交通 / 美食 / 酒店 / 门票 / 天气 / 提醒卡片
- 高信息密度，但路线必须清楚

图片好看是第二优先级，旅行流程可读是第一优先级。

## 角色系统

公开版本不绑定任何私人卡通 IP。

角色使用优先级：

1. 用户本次上传的角色设定图。
2. 本地私有配置：`travel-scrapbook-itinerary/private/character-profile.local.yaml`。
3. 通用旅行搭子贴纸。

也就是说，开源用户不会拿到你的私人角色。别人可以上传自己的角色图，或复制：

```text
travel-scrapbook-itinerary/references/character-profile.example.yaml
```

然后创建自己的本地私有角色配置。

## 私有资产保护

以下内容默认不进入 Git：

- 私人角色图
- 真实旅行行程截图
- 已生成的私人样图
- 第三方风格参考图片
- 私有角色配置
- 测试过程记录

开源前可检查：

```powershell
git status --short --ignored
git check-ignore -v travel-scrapbook-itinerary\private\character-profile.local.yaml
git check-ignore -v travel-scrapbook-itinerary\assets\character-references\your-character.png
```

## 安装方式

把 `travel-scrapbook-itinerary` 文件夹放到 Codex 可识别的 skills 目录，或在当前工作区中通过 Codex 明确使用：

```text
Use $travel-scrapbook-itinerary ...
```

示例：

```text
Use $travel-scrapbook-itinerary，把下面这段旅行计划做成 9:16 旅行流程手账图。
```

## 适合的请求

```text
我给你一个行程，先不要生图，先拆解每天路线和页数。
```

```text
我只给你目的地和日期，你帮我查天气、安排路线、推荐住宿区域和小吃，然后生成手账提示词。
```

```text
这张手账整体满意，只想把 P1-IMG3 换成我上传的照片。
```

## 不适合的请求

- 只想要普通文字旅游攻略。
- 不需要路线、时间、交通、酒店、提醒，只想做纯拼贴海报。
- 要求编造不存在的景点、车次、酒店或门票。

## 开源建议

推荐开源内容：

- Skill 规则
- 提示词模板
- 结构化拆解逻辑
- 实时查询规则
- 可替换图片槽位规则
- 通用角色占位逻辑

不要开源：

- 私人角色 IP
- 真实行程截图
- 含私人角色的生成图
- 第三方参考图原图
- 平台账号、Cookie、API Key

更多规则见：

```text
travel-scrapbook-itinerary/references/open-source-ip-policy.md
```
